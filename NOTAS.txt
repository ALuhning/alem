-- Pontos de atencao --
- O escopo local sempre vai ter prioridade sobre o global. Prefira usar local para evitar conflitos no global. Ex:

const Link = styled.div`...`; // nao recomendado, ja que pode dar conflito e ser reescrito em caso de outro componente
// fazer o mesmo

const MyComp = () => {
  const Link = styled.div`...`; // é melhor aqui do que fora do metodo MyComp
}

- Usar "componentProps" (ou outro nome) para referenciar as props de um componente. "props" já é usado globalmente
pelo ambiente BOS.

-- Pontos positivos --
Rotas: recurso interno para tratar rotas
Render: Velocidade de renderização já que todo o bundle to app está disponível em apenas um arquivo (70% mais rapido)
Import/Export: Recurso suportado
Plugins: Para inserir funcionalidade de terceiros (deve ser melhor trabalhado o gerenciador de pacotes)

-- Pontos negativos em relacao ao modo usual --
Rotas: Quando muda de pagina, o app inteiro é recarregado já que é um bundle único
Render: Fica mostrando o circulo preto de carga para carregar o bundle (isso é padrao do BOS)
